# v4.3.1 Release Notes

## Summary
This release includes important bug fixes that improve the stability of the timer loop and safely handle errors that occur during timer expiration.

## Fixed
- Added error handling during timer expiration. Errors thrown by "expired" event listeners are now caught and correctly emitted as an "errored" event, preventing the timer loop from unexpectedly crashing.
- Changed the timer loop implementation from `setInterval` to a recursive `setTimeout`. This ensures the next cycle only starts after the current cycle has fully completed, preventing overlapping executions.
