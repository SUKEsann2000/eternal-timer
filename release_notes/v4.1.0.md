# v4.1.0 Release Notes

## Summary
This release includes updates to event names, import path fixes, and internal improvements. The change in `TimerEvents`' names might be a breaking change for some users.

## Added
- Improved internal error messaging with a new `throwMessage` enum for better clarity and maintainability.

## Changed
- `TimerEvents` names have been updated: `started` is now `created`, and `stopped` is now `removed`. This change provides more accurate descriptions of the lifecycle events.

## Fixed
- Resolved import path issues (`src/` to `../`) to improve module resolution and compatibility.

## Removed
- This section is omitted as there were no removals in this release.

## Breaking Changes
- The event names for `TimerEvents` have been changed from `started` to `created` and `stopped` to `removed`. Users relying on the previous event names will need to update their code to use the new names.

## Migration Notes (if applicable)
- If you were listening to `started` or `stopped` events on `TimerEvents`, please update your event listeners to use `created` and `removed` respectively.

## Notes
- No additional notes for this release.
