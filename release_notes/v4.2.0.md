# v4.2.0 Release Notes

## Summary
This release introduces new features for controlling and monitoring the timer checking process, along with improved event handling for timer lifecycle.

## Added
- Added `checkStop()` method to programmatically stop the timer monitoring process.
- Introduced `isBusy` getter to determine if the TimersManager is actively checking for expired timers.
- New `started` and `stopped` events are now emitted by the `TimersManager` when the timer checking process begins and ends, respectively.

## Changed
- The `checkStart()` method now returns a `Promise<void>` and manages its internal interval, simplifying its usage by removing the need for manual `clearInterval` calls.
- Updated `README.md` to reflect the new `checkStop()` and `isBusy` additions.

## Fixed
- Nothing

## Removed
- Nothing

## Breaking Changes
- `checkStart()` no longer returns a `NodeJS.Timeout` object. If you were previously storing and using this ID with `clearInterval()`, you should now use the new `checkStop()` method.
