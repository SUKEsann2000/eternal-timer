# v4.3.0 Release Notes

## Summary
This release introduces improved input validation for timer lengths and adjustments, sets a minimum Node.js version, and simplifies the event system by removing several lifecycle events.

## Added
- Added strict validation for `length` in `createTimer` to ensure it is a finite number.
- Added strict validation for `delay` in `adjustRemainingTime` to ensure it is a finite number.

## Changed
- The `engines` field now requires Node.js `>=18.0.0`.

## Removed
- Removed the `interval`, `created`, `removed`, and `updated` events.

## Breaking Changes
- **Event System**: The `interval`, `created`, `removed`, and `updated` events are no longer emitted. Any logic depending on these events will break and should be removed or refactored.
- **Node.js Version**: The minimum required Node.js version is now `18.0.0`.
