# v2.3.0 Release Notes

## Summary
This release significantly enhances the `createTimer` function with increased flexibility for `JSONLTimersManager`, introduces robust generic types to `TimersManager` and its subclasses, and includes various bug fixes for improved stability.

## Added
- **Flexible `createTimer` for `JSONLTimersManager`:** The `createTimer` method in `JSONLTimersManager` now supports passing an object as an argument, allowing direct specification of `length`, `title`, and `description`. The previous behavior of passing only `length` as a number is still fully supported.
- **Generic Types:** The core `TimersManager` and its subclasses have been refactored to utilize generic types, providing enhanced type safety and improved extensibility for developers.
- **Type Definitions:** Added import and require types to improve module compatibility across different JavaScript environments.

## Changed
- **`createTimer` Argument Handling:** The internal handling of arguments for `createTimer` is now more explicit and robust, guided by the `CreateTimerOptions` type definition.

## Fixed
- Corrected an issue where `length` was not properly defined in `JSONLTimersManager`'s `createTimer` implementation when `length` was passed as a direct number.
- Addressed various type-related bugs and made minor indentation fixes.
- Updated `README.md` for improved accuracy.
- Resolved an issue with the `npm run` command.
- Fixed a typo, replacing "JSON" with "JSONL" in a storage type reference.
- Removed an unnecessary warning message in `createTimer`.

## Migration Notes
- For `JSONLTimersManager`, you can now pass an object to `createTimer` to specify `length`, `title`, and `description` more explicitly. The previous method of just passing a `number` for `length` is still fully supported, so no mandatory migration is required unless you wish to leverage the new object-based arguments for more detailed timers.

```typescript
// Old (still supported for JSONLTimersManager)
await manager.createTimer(5000);

// New (for JSONLTimersManager)
await manager.createTimer({ length: 5000, title: 'My Timer', description: 'A new timer' });

// PlainTextTimersManager still accepts only a number for length
await plainTextManager.createTimer(3000);
```
