# v2.4.0 Release Notes

## Summary
This release significantly refactors timer management, implements a robust promise-queue based concurrency control, and streamlines internal caching, resulting in improved stability, readability, and data integrity.

## Added
-   **Promise-Queue based Concurrency Control:** A robust promise-queue based mechanism has been implemented to serialize I/O operations on timer files, effectively preventing race conditions and ensuring data integrity.
-   **Adjust Remaining Time:** New functionality to adjust the remaining time of timers has been added.

## Changed
-   **Improved Type Definitions:** `Timer` type definitions have been refined for improved clarity and structural consistency.
-   **Enhanced Validation and Error Handling:** Timer validation and error handling within `TimersStore` classes have been significantly strengthened for more robust behavior.
-   **Asynchronous Callback Execution:** The `checkTimers` method now properly awaits callback execution, improving reliability and responsiveness. Documentation for this behavior has also been updated.
-   **TimersManager Refactoring**: The core logic for timer processing has been centralized within the `TimersManager` class, reducing redundancy and improving modularity. Subclasses like `JSONLTimersManager` and `PlainTextTimersManager` now primarily handle format-specific interactions.
-   **TimersStore Introduction**: Timer persistence concerns have been extracted into a new `TimersStore` component, with `JSONLTimersStore` and `PlainTextTimersStore` providing specific storage implementations. This separation enhances maintainability and allows for easier integration of new storage formats.
-   **`checkTimers` Callback Flexibility:** The `checkTimers` method now accommodates callback functions that return `void` in addition to `Promise<void>`, enhancing flexibility without introducing breaking changes.
-   **Zero-Length Timers Supported:** Creating timers with a duration of 0 seconds is now allowed and will no longer throw an error, providing more flexibility for immediate timer execution.
-   **Consistent Terminology:** Internal references using "alarm" have been standardized to "timer" for greater consistency across the codebase.

## Fixed
-   **Syntax Validation:** Issues related to `checkTimerfileSyntax` have been corrected.(using validate in uuid lib)
-   **Test Stability:** Various test failures and script-related issues have been resolved, improving overall test suite stability.
-   **PlainText Implementation:** Minor fixes have been applied to the `PlainText` implementation for better accuracy.
-   **Remaining Time Check:** A check has been implemented when adjusting remaining time to prevent unintended behavior.

## Removed
-   **Unused Code:** Unused options and obsolete `Timer` type definitions have been removed, reducing codebase bloat.
-   **Configuration Cleanup:** `cache` and `incremental` settings have been removed from `tsconfig.json` files, simplifying project configuration.

## Breaking Changes
-   **Timer File Location Restricted:** For security reasons, timer files must now reside within the project's root directory. Attempting to create or access timer files outside this directory will result in an error.
